# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

TiSMeD RDF converter: AWK scripts that convert tissue-specific gene/protein expression data from [TiSMeD](https://www.bio-add.org/TiSMeD/) (TSV format) into RDF Turtle files. The `output/` directory is large — avoid reading its contents.

## Commands

### Step 1: Assign tissue IDs (run only when `tismed_tissue_hierarchy.tsv` changes)
```sh
awk -f scripts/tissue_hierarchy_ids.awk data/tismed_tissue_hierarchy.tsv > data/tismed_tissue_ids.tsv
```

### Step 2: UBERON mapping (semi-manual, done with EBI OLS MCP)
The result is stored as `data/tismed_tissue_uberon_mapping.tsv`. Re-run only if the tissue hierarchy changes.

### Step 3: Generate ontology TTL
```sh
awk -f scripts/generate_ontology.awk data/tismed_tissue_uberon_mapping.tsv data/tismed_tissue_hierarchy.tsv > output/tismed_ontology.ttl
```

### Step 4: Convert main data files (input TSVs from TiSMeD download, not in this repo)
```sh
awk -f scripts/convert_tsgs.awk data/tismed_tissue_uberon_mapping.tsv TSGs.tsv > output/TSGs.ttl

awk -f scripts/restore_tsps.awk TSPs.tsv > TSPs_restored.tsv
awk -f scripts/convert_tsps.awk data/tismed_tissue_uberon_mapping.tsv TSPs_restored.tsv > output/TSPs.ttl
```

## Architecture

### Data files in `data/`

| File | Description |
|------|-------------|
| `tismed_tissue_hierarchy.tsv` | Source: Table S5 of the TiSMeD paper. Three-level hierarchy: System → Tissue → Sub-tissue. Uses `-` for absent levels. Trailing spaces must be stripped. |
| `tismed_tissue_ids.tsv` | Generated by `tissue_hierarchy_ids.awk`. Maps `TISMEDT_XXXXXX` IDs to tissue labels (sequential, order determined by first appearance in hierarchy). |
| `tismed_tissue_uberon_mapping.tsv` | Extends `tismed_tissue_ids.tsv` with UBERON IDs and labels (columns: ID, TiSMeD label, UBERON ID, UBERON label). Created semi-manually using the EBI OLS MCP server. |

### RDF namespaces
- `tismedo:` → `http://med2rdf.org/ontology/med2rdf/tismed/` (classes, properties, tissue instances)
- `ensg:` → `http://rdf.ebi.ac.uk/resource/ensembl/` (genes)
- `uniprot:` → `http://purl.uniprot.org/uniprot/` (proteins)
- `obo:` → `http://purl.obolibrary.org/obo/` (UBERON terms, using `_` separator)

### Script responsibilities

**`tissue_hierarchy_ids.awk`**: Scans all three columns of the hierarchy, assigns sequential `TISMEDT_XXXXXX` IDs to each unique non-`-` label in order of first appearance.

**`generate_ontology.awk`**: Two-pass. First file (`tismed_tissue_uberon_mapping.tsv`): emits `owl:Class` triples with `rdfs:label` and optional `skos:closeMatch` to UBERON. Second file (`tismed_tissue_hierarchy.tsv`): emits `rdfs:subClassOf` triples for the tissue tree.

**`convert_tsgs.awk`**: Two-pass. First file builds `tissue_id[label]` lookup from the mapping TSV. Second file (TSGs.tsv, skipping header) emits one blank-node `GeneExpressionSpecificity` per row. A single Ensembl gene ID column may contain multiple IDs separated by `"; "` — each gets its own triple set. Validates the `Tissue-specific protein` column (`yes`/`no` → `true`/`false`); exits with error on unexpected values.

**`restore_tsps.awk`**: Fixes column misalignment in TSPs.tsv where rows starting with an Ensembl ID in column 2 have columns 2–9 shifted. Detected by `$2 ~ /^ENSG/`.

**`convert_tsps.awk`**: Same two-pass pattern as `convert_tsgs.awk` but for proteins (UniProt IDs, `ProteinExpressionSpecificity`). No multi-value column handling needed.
